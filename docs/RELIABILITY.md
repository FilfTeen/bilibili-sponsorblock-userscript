# 误差与可靠性说明

## 这份脚本可靠的部分

### 较可靠

- SponsorBlock 已收录的片段跳过
- SponsorBlock 已收录的整视频 `full` 标签
- 评论 payload 明确给出的属地字段
- **维护操作执行**：v0.3.6 引入的二阶段确认和色彩预警极大降低了执行错误维护指令的可能性。

### 中等可靠

- 整视频标签接口返回的标签
- 首页 / 搜索 / 推荐 / 历史卡片的整视频胶囊展示
- 评论区商品卡广告识别

### 启发式、存在误差

- 本地页面线索推断出的商单/导流标签
- 评论区和动态页广告文案识别
- 本地学习后的商业意图修正

## 可能出现的误差类型

- 误判：把普通内容判断成广告或商单
- 漏判：未能识别实际商业内容
- 阶段性不一致：标题胶囊、缩略图胶囊、本地评论线索在短时间内未完全同步
- 页面兼容性误差：Bilibili 调整 DOM 或组件结构后，一部分增强暂时失效

## 为什么不能把它当成绝对判断

这份脚本并不掌握：

- 创作者真实商业合作合同
- 平台内部审核标记
- 用户真实地理位置
- 平台完整内容上下文

它只能基于：

- SponsorBlock 社区数据
- 页面公开信息
- 评论与动态中的文本线索
- 本地配置与学习记录

所以它的定位始终是“辅助判断工具”，不是权威裁决器。

## 推荐的使用策略

- 对评论和动态过滤先用 `仅标记`
- 对本地判断出的整视频标签保持谨慎
- 对于争议性内容，以实际视频、评论上下文和作者说明为准
- 当脚本判断与你的理解冲突时，不要盲信脚本

## 用户需要特别注意的场景

- 含品牌露出但不是商单的内容
- 开箱、测评、活动记录、免费借测、媒体体验会
- 评论区玩梗、引用、反讽、二次创作
- 页面返回中没有公开属地字段的评论回复

## 自学习机制的边界

本地学习只影响你本机这份脚本的辅助判断，不代表：

- SponsorBlock 社区共识
- Bilibili 官方标记
- 对其他用户可复现的结果

如果你错误地保留或忽略了某条本地判断，脚本后续也会沿着这个本地记录工作，直到你手动纠正。
