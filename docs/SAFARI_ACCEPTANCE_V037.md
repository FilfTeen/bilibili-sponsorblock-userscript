# Safari 验收清单 v0.3.7

这份清单专门用于 `v0.3.7` 的生态净化能力验收。

适用范围：

- `MBGA` 网络净化
- `MBGA` UI 补丁
- `MBGA` 行为解锁
- 对 BSC 既有功能的回归检查

## 验收前提

真实验收环境必须同时满足：

- Safari 主窗口
- 已登录真实账号
- 已安装当前构建产物 `dist/bilibili-sponsorblock.user.js`
- Tampermonkey 脚本已开启

不作为最终验收结论的环境：

- Chrome / Edge
- Playwright 的 Chrome smoke
- Safari 自动化窗口
- 未登录游客态的单次结果

自动化结果只能作为辅助证据，不能替代主窗口人工验收。

## 本轮必须采样的页面

1. 视频页
2. 动态页
3. 专栏页
4. 首页 / 推荐流
5. 直播间

每类页面至少保留 1 张截图。
至少保留 1 段录屏，覆盖一个完整交互链路。

## 页面级验收项

### 1. 视频页

目标：

- 遥测 / 追踪请求被压制
- SponsorBlock 主链路无回归
- 视频裁切模式正常
- 紧凑顶栏、标题胶囊、评论增强无回归

必查项：

- 片段跳过仍然工作
- 标题胶囊仍然工作
- 右侧缩略图标签仍然工作
- 评论区增强仍然工作
- 视频裁切模式按钮只出现一次
- 开关裁切模式后 `body[video-fit]` 行为正确
- 页面未出现明显错位、遮挡、点击失效

证据：

- 视频页静态截图
- 视频裁切切换前后截图
- 录屏一段，覆盖进入页面、切换裁切、播放、触发一个 BSC 原有能力

### 2. 动态页

目标：

- 宽屏模式可用
- 商品动态净化有效
- 布局未被打坏

必查项：

- “宽屏模式”开关出现且只出现一次
- 默认宽屏状态正确
- 点击后 `WIDE_OPT_OUT` 行为正确
- 商品动态或明显广告卡片被净化
- 非广告动态未被误伤
- 页面主要布局、滚动、交互正常

证据：

- 动态页截图 1 张
- 宽屏开关前后截图各 1 张

### 3. 专栏页

目标：

- 复制限制被解除
- 专栏阅读体验未受破坏

必查项：

- `.article-holder` 不再保留 `unable-reprint`
- 文字可以正常复制
- 正文排版、滚动、图片、链接交互正常

证据：

- 专栏页截图 1 张
- 复制操作录屏 1 段或至少 1 张选中文本截图

### 4. 首页 / 推荐流

目标：

- 已知广告提示和广告位被净化
- 正常卡片不被误删

必查项：

- `adblock-tips` 类提示不再干扰
- 明显广告跳转位被净化
- 正常视频卡片仍可见
- 布局没有塌陷、空白异常、点击异常

证据：

- 首页截图 1 张
- 如发现疑似误伤，额外补 1 张问题截图

### 5. 直播间

目标：

- 干扰层处理有效
- PCDN / 直播链路策略未破坏播放

必查项：

- 播放器正常加载
- 直播画面正常
- 干扰层未遮挡核心观看区域
- 清晰度或播放链路没有明显异常

证据：

- 直播间截图 1 张
- 如涉及画质/播放回退，补 1 段录屏

## 全局回归项

这部分必须在至少一个视频页样本上确认：

1. SponsorBlock 片段跳过正常
2. 标题商业标签正常
3. 缩略图标签正常
4. 评论区广告识别正常
5. 评论区属地显示未被打坏
6. 动态页商业识别正常
7. 设置面板能正常打开
8. MBGA 开关不会导致脚本整体失效

## 网络净化验收要求

本轮只接受“白名单式命中”的判断，不接受“感觉没发出去”的主观结论。

需要明确记录：

- 页面 URL
- 登录状态
- 命中的 host / path
- 是否属于 `cm.bilibili.com` / `data.bilibili.com` / 已知 reporter
- 是否有误伤正常接口的迹象

如果看到了接口失败，先判断它是不是：

- 登录态差异
- 用户权限差异
- 页面 AB 实验
- 站方临时异常

不能先入为主地归因为脚本 bug。

## 通过标准

`v0.3.7` 的生态净化验收通过，至少应满足：

- 自动化检查全绿：`check / test / build / validate:safari`
- Safari 主窗口人工采样覆盖 5 类页面
- 没有发现 SponsorBlock 主链路回归
- 没有发现明显页面结构错乱
- 没有发现新增的高侵入副作用
- 剩余问题都已被记录为明确的残余风险，而不是“未知异常”
